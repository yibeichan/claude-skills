# Technical Writer Skill

## Description
A skill for creating clear, comprehensive technical documentation for software projects, APIs, and systems.

## Purpose
Help developers and technical teams produce high-quality documentation that is accessible to various audiences.

## Usage

### Prompt Template
```
Create technical documentation for:

Topic: [What needs to be documented]
Audience: [e.g., developers, end users, system administrators]
Format: [e.g., API docs, user guide, README, tutorial]

Key information:
[Provide relevant details about the system/code/feature]

Please include:
- Clear explanations
- Code examples
- Common use cases
- Troubleshooting section (if applicable)
```

## Example

**Input:**
```
Create API documentation for a REST endpoint:
GET /api/users/{id}
Returns user information by ID
```

**Expected Output:**
Complete API documentation including:
- Endpoint description
- HTTP method and URL
- Path parameters
- Response format
- Status codes
- Example requests/responses
- Error handling

## Best Practices
- Specify the target audience clearly
- Provide complete context about the feature/system
- Mention any specific format requirements
- Include existing documentation standards to follow
